> TRD lets worldwide investors invest in productive assets in East Africa,
> starting with electric motorbikes, via the TRD Open-Ended Adaptive Debt
> Facility. TRD provides the trust and accounting layer for private credit — a
> live Risk Monitor, updated daily from independent third-party data: the
> micro-entrepreneurs' real-world productivity from Bolt, and the lender's
> aggregated loan book from their Loan Management System.

TRD's first demand side customer / operating partner / borrower is
[Jiwambe](https://www.jiwambe.com/), a Nairobi-based asset-financing and
fleet-management company with +1000 active electric motorcycles
serving gig workers and micro-entrepreneurs.

TRD has two core product components:

1. The **[Risk Monitor](https://therealdeal.app/)** turns borrower and
   third-party operating data into investor-readable evidence about the live
   loan book.
2. The **[Open-Ended Adaptive Debt Facility interactive model](https://therealdeal.app/docs/model)**
   explains how the proposed facility behaves across rates, liquidity,
   withdrawals, repayments, and stress scenarios.

This is TRD's [Sui Overflow 2026](https://overflow.sui.io/) submission for the
**DeFi & Payments** track. The [strategic roadmap](./strategic_roadmap.md)

## Public Repository Scope

The following TRD repositories are private and are not shared as part of this
submission:

- **Knowledge base:** compound-engineering knowledge, market research, legal
  research, company documentation, and the AI advisory system.
- **Backend:** integrations with the first borrower and the relevant third-party
  data providers: Bolt, the borrower's Loan Management System, M-PESA, and the
  NTSA vehicle registry.
- **Frontend:** the Risk Monitor, TRD Docs, the front-end explainer and
  interactive model for the TRD Open-Ended Adaptive Debt Facility, and other
  TRD product interfaces.
- **Smart contracts:** the Sui Move workspace for the future on-chain facility;
  this repository is not shared.
- **MVP collection:** the complete sequence of internal technical experiments
  covering the TEE, Nautilus, Seal, Walrus, consent, and verification stack;
  this repository is not shared.

This public repository contains the Overflow submission documents and an
[explainer for the deployed Sui package ID](./deployed_package.md). It also
references the separate
[MVP Demo — Sui Overflow Hackathon 2026 repository](https://github.com/ufahamu-labs/mvp_demo_sui_overflow_hackaton_2026),
which contains a reduced MVP prepared for the hackathon to explain TRD's trusted
data pipeline.

The
[TRD Open-Ended Adaptive Debt Facility Interactive Model](https://therealdeal.app/docs/model)
was built to help the first operating partner (borrower) understand and discuss
the proposed facility.
